🍽️ Smart Dining Recommender (프로젝트명)
제주도 런케이션(Learncation) 기간 중에 진행할 개인 프로젝트.
사용자의 위치와 취향을 기반으로, 실패 없는 맛집을 찾아주는 머신러닝 기반 추천 서비스

1. 프로젝트 제작 배경
네이버 지도나 구글 지도와 같은 기존 지도 서비스는 별점과 리뷰 개수 등 정량적인 정보는 제공하지만, 사용자의 구체적인 니즈를 한 번에 충족시키기에는 한계가 있었습니다.

기존 서비스의 문제점:

- "별점이 높으면서 리뷰도 많은 곳", "한식/양식 등 특정 카테고리", "술집 vs 밥집", "특정 분위기" 등 복합적인 조건으로 검색하기 어렵습니다.

- 검색된 식당들을 일일이 클릭하여 리뷰를 확인해야 하는 번거로움이 존재합니다.

- LLM(ChatGPT 등)처럼 대화형 추천이나 스마트한 필터링 기능이 지도 앱 내에 부재합니다.

=> "사용자가 원하는 조건의 식당을 직관적으로 추천받을 수 없을까?" 라는 고민에서 출발하여, 위치 기반의 스마트 맛집 추천 서비스를 기획하게 되었습니다.

2. 프로젝트 목표
사용자가 입력한 주소를 기준으로 설정된 반경(N km) 내의 맛집을 분석하여 추천해주는 서비스를 구현합니다.
단순 나열이 아닌, 머신러닝을 통해 사용자가 선호할 만한 최적의 장소를 제안하는 것을 목표로 합니다.

4. 주요 기능 
- 위치 기반 검색: 도로명 주소 입력 시, 해당 위치를 기준으로 반경 N km 내의 식당을 탐색합니다.

- 스마트 필터링: 별점, 리뷰 수, 음식 종류(한/중/일/양식), 식당 분위기, 주류 판매 여부 등을 종합적으로 고려합니다.

- 머신러닝 추천: Python Scikit-learn을 활용하여 수집된 데이터를 학습하고, 사용자 취향에 맞는 식당을 우선순위로 추천합니다.

- 시각화: Google Maps API를 연동하여 추천된 식당의 위치를 지도상에 직관적으로 표시합니다.

4. 기술 스택 (Tech Stack)
Frontend
- HTML
- CSS
- JS

Backend & Machine Learning
- python
- Scikit-learn
- Flask

API
- Google Maps API

5. 기대 효과 
검색 시간 단축: 사용자가 일일이 리뷰를 확인하는 시간을 획기적으로 줄여줍니다.

만족도 향상: 단순 인기순이 아닌, 분위기와 목적(식사/음주)에 맞는 장소를 추천하여 방문 실패 확률을 낮춥니다.

기술적 성장: 프론트엔드와 데이터 머신러닝(Scikit-learn)을 결합하여 실제 서비스를 구축해보는 경험을 쌓습니다.
